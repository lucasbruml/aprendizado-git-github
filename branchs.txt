O que são Branches?

Branches são ramificações separadas do código principal em um repositório Git. Cada branch representa uma
linha independente de desenvolvimento, permitindo trabalhar em novas funcionalidades, correções de bugs
ou experimentos sem interferir no código principal.

Por que Usar Branches?

Isolamento: Permite desenvolver novas funcionalidades ou correções sem afetar o código estável.

Experimentação: Facilita testar novas ideias sem comprometer o trabalho existente.

Colaboração: Equipes podem trabalhar simultaneamente em diferentes funcionalidades e depois mesclar suas 
mudanças de volta ao código principal.

Operações Básicas com Branches

Criar: git branch nome-da-branch

Mudar para uma Branch: git checkout nome-da-branch

Criar e Mudar para uma Nova Branch: git checkout -b nome-da-branch

Listar Branches: git branch

Mesclar uma Branch: git merge nome-da-branch

Deletar uma Branch: git branch -d nome-da-branch

Boas Práticas

Nomenclatura Descritiva: Escolha nomes claros e significativos para suas branches.

Manter Limpo: Exclua branches após integrar suas mudanças no código principal.

Integração Contínua: Mesclar frequentemente suas branches de volta ao código principal para evitar 
conflitos.

Conclusão

Branches são cruciais para um desenvolvimento organizado e colaborativo no Git, permitindo flexibilidade 
no gerenciamento de versões e no trabalho simultâneo em diferentes funcionalidades do projeto.
